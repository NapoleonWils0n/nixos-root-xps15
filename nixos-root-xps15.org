**** build only

Dry-Run (Build only):

This builds the configuration and shows you exactly what would change without actually applying it.

#+begin_src sh
doas nixos-rebuild build --flake '.#pollux'
#+end_src

This creates a result link in your current directory.

If it finishes without errors, your config is syntactically and structurally correct.

**** dry-activate

This builds the config and shows the changes that would occur (like service restarts) without actually switching.

#+begin_src sh
doas nixos-rebuild dry-activate --flake '.#pollux'
#+end_src

**** test

This applies the configuration but does not add it to your boot menu.

If something goes wrong, a simple reboot will bring you back to your previous working state.

#+begin_src sh
doas nixos-rebuild test --flake '.#pollux'
#+end_src

**** nixos-rebuild

run nixos-rebuild switch with the flake option

#+begin_src sh

#+end_src

**** git commit

add the new flake to the git repo and push

#+begin_src sh
git add .
git commit -m "nixos flake"
git push github master
#+end_src

if there are any errors you can revert the flake.lock file

#+begin_src sh
git checkout 'HEAD^' -- flake.lock
#+end_src

This command will revert flake.lock to its state in the previous commit.

If you updated multiple times, you might need to go back further
e.g., HEAD~2 for two commits ago, or look at

#+begin_src sh
git log --oneline flake.lock
#+end_src

to find the exact commit hash
